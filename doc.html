리액트와 일반적인 웹과의 가장 주된 차이점

  • 싱글페이지 애플리케이션
  • 기존의 웹은 이동할 때 화면의 모든 요소가 업데이트
  • 리액트는 변화하는 부분만 업데이트

npm과 npx의 차이점

  • npx는 1회 설치용, 한 번만 설치되면 다시 관리할 필요 없는 요소에 사용
  • npx create-react-app

리액트의 요소

  • JSX - HTML with JavaScript

    • {}을 이용해서 JS요소 첨가 가능

  • class based, functional based

    • 현재 standard는 functional

  • dynamic change를 지원해 개발을 편하게 함

  • package.json

    • 이름
    • 버전
    • dependencies - 사용하고 있는 라이브러리
    • scripts - npm으로 실행할 수 있는 스르크립트들
    • eslintConfig
    • browserslist

  • package-lock.json

    • dependencies의 exact 버전 확인 가능

  • index.js

    • 메인 프로그램
    • 컴포넌트를 조합해 실제 렌더링

  • App.js

    • root component
    • 화면에 표시되는 부분 정의

  • index.html

    • ndex.js를 렌더링한 결과물

리액트 컴포넌트 만들기

  • 리액트는 컴포넌트로 구성
  • 컴포넌트는 리턴으로 하나의 html 태그를 반환
  • div로 간단하게 가능하지만 css속성을 배제한 채로 리턴하려면 React.Fragment를 사용

useState

  • 컴포넌트에서 동적인 값(State)을 관리
  • react의 default export가 아니기 때문에 import { useState } 형태로 작성
  • const [state, setstate] = useState(initialState)
  • state = 변수명, setstate = 변수 관리 함수, initialState = 초기값
  • 페이지 리로딩 없이 변수가 사용되는 부분의 DOM만 재구성
  • 전체 페이지를 리로딩하면 state값이 초기값으로 바뀜
